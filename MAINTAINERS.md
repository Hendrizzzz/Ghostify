# Maintainers

Ghostify is currently maintained by one primary maintainer.

| Maintainer | Role | Responsibilities |
| --- | --- | --- |
| [@Hendrizzzz](https://github.com/Hendrizzzz) | Primary maintainer | Runtime architecture, privacy and security decisions, issue and pull-request review, browser compatibility, releases, store submissions, and public verification |

## Decision process

Changes are reviewed against Ghostify's privacy scope, compatibility risk, test
evidence, and maintenance cost. Privacy-sensitive runtime, permission, and
release changes require the primary maintainer's review. Community proposals
and pull requests are welcome, but the primary maintainer remains accountable
for final decisions and published packages.

The repository's `CODEOWNERS` file records this current review ownership. It is
not evidence of a larger team. If maintainer responsibilities change, this file
and `CODEOWNERS` must be updated together.

## Continuity

The single-maintainer model is a project risk. To reduce it, Ghostify keeps its
build, validation, release, browser-distribution, architecture, and public
verification processes in the repository. A future maintainer must be given
explicit repository and store access before being listed here.
